About this Template

The Store Catalog template is comprised of a Bluemix backend (you're here!) and iOS application that serve to demonstrate a common and reusable design pattern using new technology from IBM's Bluemix portfolio.

The title screen of the iOS application is called Store Catalog. This mobile application allows users to browse, filter by category, and see the price, rating, description, etc. of each article of clothing that is available from the store.

The iOS application utilizes a MVC software architectural pattern and connects to our Bluemix backend to retrieve and filter any piece of data it needs by using the API Connect, Cloudant, and Object Storage services that are provided by IBM.

The following sections explain what each of these services do, how they are implemented in this backend, and where to get more detailed information about the services from the official documentation.


  • API Connect

    IBM API Connect is a comprehensive end-to-end API lifecycle solution that enables the automated creation of APIs, simple discovery of systems of records, self-service access for internal and third party developers and built-in security and governance. Using automated, model-driven tools, create new APIs and microservices based on Node.js and Java runtimesâ€”all managed from a single unified console.

    In our template, we use API Connect to connect to our Cloudant datasource and expose an endpoint to our custom Object Storage module to get images of the clothing items. API Connect gives us a unified communication platform for our mobile application.

    Click the View API Reference button to browse the API we've constructed for the Store Catalog template. For more information about API Connect, see the official documentation below.

  • Cloudant NoSQL DB

    Cloudant NoSQL DB provides access to a fully managed NoSQL JSON data layer that's always on. This service is compatible with CouchDB, and accessible through a simple to use HTTP interface for mobile and web application models.

    In this template, while running the bluegen tool, we provision a Cloudant NoSQL DB service instance and supply that with data from a products.json file which contains the data from the Store Catalog.

    If you open the Cloudant NoSQL DB service instance, you can browse the data inside the template. Each document contains information about the price, rating, description, etc. of each article of clothing that is available from the store.

    For more information about Cloudant NoSQL DB, see the official documentation below.

  • Object Storage

    Object Storage provides an unstructured cloud data store to build and deliver cloud applications and services with lowered cost, reliability, and speed to market. Bluemix developers and users can access and store unstructured data content and can interactively compose and connect to applications and services. The Object Storage service also provides programmatic access via API, SDKs and a consumable UI for object management.

    In this template, while running the bluegen tool, we provision a Object Storage service instance and supply that with data from a clothes folder which contains the data from the Store Catalog.

    If you open the Object Storage service instance, you can browse the data inside the template. The clothes container contains the images of each item available from the store.

    For more information about Object Storage, see the official documentation below.
